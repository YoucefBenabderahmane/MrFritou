Application Overview
What the app is, who uses it, and its core purpose

🍽️ What is Daily Dose Restaurant OS?

Daily Dose Restaurant OS is a complete digital restaurant management platform built as a web application. It serves two main audiences simultaneously: restaurant staff and administrators (who manage orders, menu, finances, and staff) and customers (who browse menus and place takeaway orders from their phones). The system operates in real time, with the admin dashboard auto-refreshing order data every 3 seconds.

👨‍💼 Admin / Staff Interface

A protected dashboard accessible only after login. Manages the full restaurant lifecycle: incoming orders, menu configuration, inventory, staff payroll, and financial reporting. Supports roles: Admin (full access) and Manager/Waiter (limited access).

📱 Customer-Facing Interface

A public, mobile-optimized interface accessible via QR code scan or direct URL. Customers browse the live menu, add items to cart with notes, enter their details, and submit takeaway orders — no login required.

Single Restaurant MVP: The current version is designed for one restaurant at a time. All data (orders, menu, settings, staff) belongs to one shared database. This MVP is the foundation for the upcoming multi-tenant SaaS transformation.

Full Feature List
All modules and capabilities of the application
📊

Live Dashboard

Real-time KPIs: monthly revenue, order count, expenses, pure profit. Recent orders table. Financial health widget. Role-based view (Admin sees financials, Managers see orders only).

🛒

Order Management

Live-polling order list (every 3s). Filter by status. Click-to-open slide panel. Update order status, adjust item quantities, remove/add items. Manual new order creation. Kitchen + payment print tickets.

🍽️

Menu Management

CRUD for menu items with category, price, stock levels, and image upload. Items marked active/low. Dishes auto-hide from customer menu when stock = 0.

📦

Stock Management

Track raw ingredients and supplies. Define name, category, quantity, unit, and unit cost. Real-time cost tracking per item. Full CRUD with quantity adjustments.

👷

Staff / HR Module

Add employees with role, salary, and payment frequency (Daily/Weekly/Monthly). Log individual payments. Activate/deactivate workers. Full payment history per worker.

📈

Financial Reports

Revenue vs. Expenses area chart. Breakdowns by Day/Week/Month/Year. Revenue sources split (orders vs. other income). Worker payroll history. PDF export with full tables. Profit margin calculation.

💳

Transaction Ledger

Manual income and expense logging. Categories for each transaction. Feeds into financial reports calculations for "other income" and "expenses" totals.

🔐

Authentication

Username/password login with JWT (httpOnly cookie). Google OAuth social login. User registration. Password hashing. Role-based access control (admin/manager/waiter). Profile editing with avatar upload.

🖨️

Thermal Printer Support

Register printers (kitchen or payment type). Connect via USB (WebUSB API) or network IP. Set default printer per type. Print kitchen tickets (items + notes) and payment receipts (logo, totals) directly from browser.

📱

Customer Takeaway App

Public landing page with restaurant branding. Category-filtered menu grid with images. Cart with item notes. Customer name + phone required for takeaway. Order confirmation modal. Fully mobile-responsive.

🔢

QR Code Generator

Auto-generates one QR code per table (e.g. Table 1 → /menu?table=Table+1). Download individual or all QR codes. Print-ready layout. Driven by tablesCount setting.

⚙️

Settings & Customization

Restaurant name, address, logo upload. 40+ currency options. 3 interface languages (English, French, Arabic with RTL). 8 color themes (Ember, Sapphire, Forest, Ruby, Cosmic, Rose Gold, Midnight, Parchment). Table count management.

Architecture & Data Flow
How the system components interact

🔄 Order Lifecycle

Customer scans QR → Browses /menu → Adds to cart → POST /api/orders → DB creates Order → Admin dashboard polls → Staff prints kitchen ticket → Status → Preparing → Ready → Completed

🔐 Authentication Flow

User submits credentials → POST /api/auth/login → Verify password (bcrypt) or Google ID token → Sign JWT (jsonwebtoken) → Set httpOnly cookie → Next.js middleware validates cookie on every request

📊 Financial Calculation Flow

GET /api/reports/stats?range=month → Prisma aggregates Orders (completed) + FinTransactions → Revenue = order totals + income transactions → Expenses = expense transactions + worker payments → Profit = Revenue − Expenses

🗄️ Database Layer

Prisma ORM generates a type-safe client from schema.prisma. A singleton client is exported from lib/db.ts to prevent connection pool exhaustion in Next.js dev mode. All tables live in one Neon PostgreSQL instance.

🌍 State Management

Zustand store (lib/store.ts) holds client-side state: currency symbol, active language, restaurant name/logo, theme preference, dark mode, current user session, and live order notifications.

Internationalization (i18n)
Multi-language support system

🌐 3-Language Support

The app supports English, French, and Arabic (with full RTL layout switching). All UI text is defined in lib/i18n.ts — a large translation dictionary with a t(key, language) helper function. The language preference is stored in RestaurantSettings in the database and loaded on every page. Arabic triggers dir="rtl" on the root element automatically using the isRTL(language) utility.

🇬🇧 English

Default language. Full coverage across all admin pages, order management, settings, and customer-facing menu.

🇫🇷 French

Complete French translation including financial terminology (Chiffre d'Affaire, commandes, etc.).

🇸🇦 Arabic (RTL)

Full Arabic translation with automatic right-to-left layout reversal applied to the entire interface.

Thermal Printing System
Kitchen and payment ticket printing via browser

🍳 Kitchen Ticket

Prints items + quantities + notes in large monospace font optimized for 80mm thermal paper. Includes order number and table/takeaway designation. Items are ALL CAPS for visibility. Auto-moves status to "Preparing" when printed.

🧾 Payment Receipt

Formal receipt with restaurant logo, name, order number, itemized list with quantities and subtotals, and total due. Formatted for 80mm thermal paper. Uses @page { size: 80mm auto; } CSS.

How it works: The print area is a hidden #print-area div rendered in the DOM. When the admin clicks "Print Kitchen" or "Print Payment", the CSS switches all body elements to visibility: hidden except #print-area, then calls window.print(). The browser print dialog shows a perfectly formatted 80mm ticket.

🖨️ Printer Registration (WebUSB)

Printers are registered in the Printer database table with their type (kitchen/payment), IP address (for network printers), or USB label (detected via the browser's navigator.usb.requestDevice() API). The WebUSB API allows direct USB printer detection from the browser without drivers.
